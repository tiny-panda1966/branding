Accessibility Guidelines

Inclusive design ensures everyone can interact with our brand. Accessibility isn't optionalâ€”it's essential. These guidelines help create experiences that work for all users.

Core Principles

Colour Contrast

Minimum 4.5:1 ratio for normal text, 3:1 for large text (WCAG AA)

Text Alternatives

All images must have descriptive alt text; icons need labels

Keyboard Navigation

All interactive elements must be accessible via keyboard

Focus Indicators

Clear visible focus states on all interactive elements

Readable Fonts

Minimum 16px body text, 1.5 line height, adequate spacing

No Colour-Only Info

Never rely solely on colour to convey information

Testing Tools
  • Contrast Checker: WebAIM Contrast Checker, Stark plugin
  • Screen Readers: VoiceOver (Mac), NVDA (Windows)
  • Keyboard Testing: Navigate using Tab, Enter, Arrow keys
  • Automated Testing: axe DevTools, WAVE, Lighthouse
  • Colour Blindness: Sim Daltonism, Color Oracle
Compliance Target

All digital materials must meet WCAG 2.1 Level AA standards. For public-sector projects, aim for Level AAA where feasible. Conduct accessibility audits quarterly and before major launches.
